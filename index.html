RI 2025 to 2025 Migration (SFTP export/import, DB migration)

This runbook assumes you will always migrate the configuration database, and that you have technician-mode access (root shell) on both appliances. File transfer is done via the sftp user and the jail at /var/sftp/files.

SFTP user sftp SFTP password idautoAdmin Bundle URL ri2025-migration-bundle.tar.gz Always migrate DB yes
Source host
SOURCE_HOST (existing appliance with data)
Target host
TARGET_HOST (new appliance with DB installed)
Bundle URL
BUNDLE_URL https://ri-docs-files.s3.us-east-1.amazonaws.com/ri2025-migration-bundle.tar.gz

Source appliance steps (create export archive in /var/sftp/files)

Run in technician mode (root shell). These commands download the bundle, extract it, then generate an export archive in the sftp jail.
Source: download and extract bundle
cd /var/tmp
rm -rf ri2025-migration-bundle ri2025-migration-bundle.tar.gz
curl -fL -o ri2025-migration-bundle.tar.gz "https://ri-docs-files.s3.us-east-1.amazonaws.com/ri2025-migration-bundle.tar.gz"
tar -xzf ri2025-migration-bundle.tar.gz
cd ri2025-migration-bundle
chmod 700 *.sh
Source: set sftp password to idautoAdmin (optional if already set)
passwd sftp
When prompted, enter idautoAdmin (twice). If your appliance uses a different supported method, use that instead.
Source: export to a predictable filename (recommended)
./export-appliance-2025.sh --out-file=ri-export.tar.gz
Source: confirm archive exists
ls -lh /var/sftp/files/ri-export.tar.gz
Next: On the target, you will pull files/ri-export.tar.gz from the source via sftp and import it.

Target appliance steps (pull archive via sftp and import, always migrate DB)

Run in technician mode (root shell). The migration command will prompt for the sftp password unless your environment supports non-interactive auth.
Target: download and extract bundle
cd /var/tmp
rm -rf ri2025-migration-bundle ri2025-migration-bundle.tar.gz
curl -fL -o ri2025-migration-bundle.tar.gz "https://ri-docs-files.s3.us-east-1.amazonaws.com/ri2025-migration-bundle.tar.gz"
tar -xzf ri2025-migration-bundle.tar.gz
cd ri2025-migration-bundle
chmod 700 *.sh
Target: optional sftp connectivity check
sftp -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null "sftp@SOURCE_HOST"
Type idautoAdmin when prompted, then run: ls and bye.
Target: migrate (always migrate DB)
./migrate-appliance-2025.sh --src-host=SOURCE_HOST --src-sftp-file=files/ri-export.tar.gz --migrate-db
When prompted for the sftp password, enter: idautoAdmin.
Target: start RI and verify
systemctl start rapididentity
systemctl status rapididentity --no-pager
journalctl -u rapididentity -n 200 --no-pager
If the target already had a DB installed, and you run with --migrate-db, expect the script to rebuild and load DB data. Use this only when the target DB is disposable and you want it replaced by the source data.

Troubleshooting shortcuts

If something fails, capture the last 100 lines of script output and the RI journal.
Disk space and archive size
df -h
ls -lh /var/sftp/files/ri-export.tar.gz
du -sh /var/opt/idauto 2>/dev/null || true
Confirm sftp jail contents (on source)
ls -lah /var/sftp/files
Service status and logs (on target)
systemctl status rapididentity --no-pager
journalctl -u rapididentity -n 400 --no-pager
DB quick checks (on target)
cat /var/opt/idauto/rapididentity.properties | grep -E '^(db\.type|db\.host|db\.database|db\.username)=' || true
Replace SOURCE_HOST with the source appliance hostname or IP. Keep the sftp password consistent across test runs to reduce friction.
